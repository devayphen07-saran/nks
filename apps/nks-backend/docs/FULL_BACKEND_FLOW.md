## 3. Entry Point & Bootstrap

**`src/main.ts`** — Middleware and security applied in this exact order:

```
1.  validateEnv()                    ← Zod schema; exits with per-field errors if invalid
2.  NestFactory.create(AppModule)    ← bufferLogs=true
3.  app.useLogger(Logger)            ← nestjs-pino structured logger
4.  helmet(...)                      ← CSP, HSTS, frameguard, referrer policy
5.  Permissions-Policy header        ← camera=(), microphone=(), geolocation=(), usb=()
6.  trust proxy                      ← configurable hops (TRUST_PROXY_HOPS env)
7.  app.setGlobalPrefix('api/v1')
8.  app.enableCors(buildCorsConfig)  ← origin whitelist from ALLOWED_ORIGINS
9.  cookieParser(cookieSecret)       ← signed cookies
10. requestIdMiddleware               ← X-Request-ID
11. CsrfMiddleware                   ← double-submit CSRF token
12. setupSwagger(app)                ← only if NODE_ENV !== 'production'
13. app.enableShutdownHooks()
14. app.listen(port)
```

**Helmet directives:**
- `defaultSrc: ['self']`
- `scriptSrc: ['self']` — no inline scripts
- `styleSrc: ['self', 'unsafe-inline']` — needed for styled-components
- `imgSrc: ['self', 'https:', 'data:']`
- `frameSrc: ['none']` — prevents clickjacking
- `objectSrc: ['none']`
- HSTS: 1-year maxAge, includeSubDomains + preload in production

## 4. App Module & DI Graph

**`src/app.module.ts`**

**Module import order** (acyclic DAG — order matters for circular prevention):

```
AppModule
├── ConfigModule.forRoot()           @Global
├── DatabaseModule                   @Global — reads ConfigModule
├── ScheduleModule.forRoot()         cron jobs
├── EventEmitterModule.forRoot()     async events
├── HealthModule                     GET /health
├── GuardsModule                     → imports AuthModule exports
├── LoggerModule (nestjs-pino)
├── AuditModule                      @Global
├── PermissionsChangelogModule       @Global
├── AuthModule                       → RolesModule → StoresModule → DatabaseModule
├── RolesModule                      → StoresModule → DatabaseModule
├── RoutesModule                     → DatabaseModule
├── LocationModule                   → DatabaseModule
├── LookupsModule                    → DatabaseModule
├── CodesModule                      → DatabaseModule
├── UsersModule                      → DatabaseModule
├── StatusModule                     → DatabaseModule
├── EntityStatusModule               → DatabaseModule
├── SyncModule                       → AuthModule (guards)
└── StoresModule                     → DatabaseModule
```

> **Critical constraint:** `RolesModule` and `StoresModule` never import `AuthModule` — this prevents circular dependency. `AuthModule` imports both of them.

**Global providers (applied to every request, in order):**

| Provider | Class | When Applied |
|----------|-------|-------------|
| `APP_FILTER` | `GlobalExceptionFilter` | All exceptions |
| `APP_PIPE` | `TrimStringsPipe` | All handlers |
| `APP_PIPE` | `ZodValidationPipe` | All handlers |
| `APP_GUARD` | `CsrfGuard` | Mutating methods (POST/PUT/PATCH/DELETE) |
| `APP_GUARD` | `AuthGuard` | All routes (default-deny) |
| `APP_GUARD` | `RateLimitingGuard` | All routes |
| `APP_INTERCEPTOR` | `TransformInterceptor` | All responses |
| `APP_INTERCEPTOR` | `TimeoutInterceptor` | All requests (30s timeout) |

## 5. Config Layer

### `env.validation.ts`

- `validateEnv()` — called in `main.ts` before `NestFactory.create()`. If any variable fails, the process exits with a per-field error list.
- `getValidatedEnv()` — alternative for test modules.

**Zod schema enforces:**
- `NODE_ENV` ∈ `['development', 'staging', 'production']`
- `PORT` coerced to number
- All HMAC secrets ≥ 32 characters
- `DATABASE_URL` is a valid PostgreSQL URL
- Rate limiting toggles: string `"true"/"false"` → boolean transform

### `app.config.ts` — `registerAs('app')`

- `nodeEnv`, `port`, `trustProxyHops`, `csrfHmacSecret`
- `allowedOrigins`: CSV string → `string[]`

### `app-config.service.ts`

Typed facade:
```typescript
get isProduction(): boolean
get isDevelopment(): boolean
get isStaging(): boolean
get port(): number
get trustProxyHops(): number
get csrfHmacSecret(): string
get allowedOrigins(): string[]
```

### `jwt.config.ts` — `JWTConfigService`

Manages RS256 key pair lifecycle:
- Loads keys from `JWT_KEYS_DIR` (default `./secrets`)
- `KeyRotationScheduler` checks key age daily (`@Cron('0 2 * * *')`)
- Grace period: 30 days — old key still validates during overlap
- `kid` (Key ID) = SHA-256 thumbprint of public key
- `/auth/mobile-jwks` endpoint serves JWKS for mobile offline verification

### `cors.config.ts` — `buildCorsConfig()`

```typescript
origin: (origin, callback) => {
  if (!origin) callback(null, true);   // Mobile/server-to-server: no Origin header
  else if (whitelist.includes(origin)) callback(null, true);
  else callback(new Error('Not allowed by CORS'));
}
credentials: true
methods: ['GET', 'POST', 'PUT', 'DELETE', 'PATCH', 'OPTIONS']
```

> No-origin requests (mobile apps) are intentionally allowed. Auth layer (JWT/session) provides the real security boundary.

// Pool config
pg.Pool {
  max: 20,
  idleTimeoutMillis: 30000,
  connectionTimeoutMillis: 2000,

- `onModuleDestroy()` + `onApplicationShutdown()` → `pool.end()`

### `base.repository.ts`

All repositories extend this:

```typescript
abstract class BaseRepository {
  constructor(@InjectDb() protected readonly db: Db) {}

  protected conn(tx?: DbTransaction): Db | DbTransaction
    // Returns tx if inside a transaction, otherwise db

  protected async paginate<T>(
    dataPromise: Promise<T[]>,
    countFactory: () => Promise<{ total: number }[]>,
    page: number,
    pageSize: number,
  ): Promise<{ rows: T[]; total: number }>
    // Auto-skips COUNT query when result set < pageSize (optimization)
}
```

### `transaction.service.ts`

Single entry point for all DB transactions:

```typescript
async run<T>(
  fn: (tx: DbTransaction) => Promise<T>,
  options: { timeout?: number; name?: string } = {},
): Promise<T>
```

- Drizzle auto-rollback on any uncaught error in `fn`
- Optional `SET LOCAL statement_timeout` (validates integer, max 1 hour)
- Logs transaction name + duration

### `query-helpers.ts`

```typescript

### RBAC Tables

#### `roles`

| Column | Notes |
|--------|-------|
| `code` | varchar(30) — SUPER_ADMIN, MANAGER, CASHIER, etc. |
| `roleName` | varchar(50) |
| `storeFk` | FK → store.id; NULL = system role |
| `isSystem` | true only for SUPER_ADMIN |
| `isEditable` | default true |

**Unique indexes:**
- `(code, storeFk)` WHERE `deleted_at IS NULL AND store_fk IS NOT NULL` — per-store custom roles
- `(code)` WHERE `deleted_at IS NULL AND store_fk IS NULL` — system roles

#### `role_permissions`

| Column | Notes |
|--------|-------|
| `roleFk` | FK → roles.id, onDelete: cascade |
| `entityTypeFk` | FK → entity_type.id |
| `actionFk` | FK → permission_action.id |
| `allowed` | boolean default false |
| `deny` | boolean default false |

**Deny semantics:** `deny=true` on any row for `(entity, action)` → ALL grants suppressed (hard block), even if other roles allow it.

**Check:** `NOT (allowed=true AND deny=true)` on same row.

#### `permission_action`

`code` ∈ `VIEW | CREATE | EDIT | DELETE | EXPORT | APPROVE | ARCHIVE`

#### `entity_type`

- `code` — INVOICE, PRODUCT, CUSTOMER, ORDER, etc.
- `defaultAllow` — if true, VIEW is granted without an explicit `role_permissions` row
- `parentCode` — hierarchy support (e.g., INVOICE_LINE → INVOICE)
- `isHidden` — hides from UI permission matrix

#### `routes`

| Column | Notes |
|--------|-------|
| `parentRouteFk` | self-ref FK, onDelete: set null |
| `routeType` | sidebar \| tab \| screen \| modal |
| `routeScope` | admin \| store |
| `isPublic` | skips AuthGuard |
| `enable` | soft toggle separate from isActive |
| `entityTypeFk` | FK → entity_type.id (permission binding) |
| `defaultAction` | default 'view' |

**Unique:** `(routePath, routeScope)` WHERE `deleted_at IS NULL`

Links roles to routes for navigation permission computation.

## 9. Common Layer — Guards

### `AuthGuard` (global, default-deny)

```
canActivate(context):
  1. Check IS_PUBLIC_KEY → if true, allow
  2. Extract token:
     - Authorization: Bearer <token> (mobile)
     - Cookie: nks_session (web)
     - If both present → 401 (mutual exclusion)
  3. Find session: AuthContextService.findSessionByToken(token)
  4. Validate: exists + expiresAt >= now + not in JTI blocklist
  5. Load user context: UserContextLoaderService.load(userFk, activeStoreFk, sessionId)
  6. Enforce account status: AuthPolicyService.enforceAccountStatus(...)
     → Checks isBlocked, accountLockedUntil
  7. Detect IP change: compare HMAC(currentIp) vs session.ipHash
     → Logs warning; does NOT block (proxy/VPN changes are common)
  8. Session rotation (cookie-based only):
     → If now - session.lastRotatedAt > ROTATION_INTERVAL_SECONDS (3600)
     → Generate new token, update session row, set new nks_session cookie
  9. Attach: req.user = SessionUser, req.session = { id, expiresAt }
```

### `RBACGuard`

```
canActivate(context):
  1. Resolve REQUIRE_ENTITY_PERMISSION_KEY metadata (method > param > class)
  2. If no metadata → allow (no permission check required)
  3. Resolve entity code:
     - Static from decorator
     - From URL param (e.g., :entityCode)
     - From @EntityResource class-level fallback
  4. Validate entity code against known registry → 400 if unknown
  5. For STORE scope:
     - Assert activeStoreId present → 403 if missing
     - Load store + verify user is owner or has role
  6. For PLATFORM scope: no store check
  7. Evaluate: permissionEvaluator.evaluate(ctx, requirement)
  8. Throw 403 if not permitted
```

### `CsrfGuard`

```
canActivate(context):
  1. Skip for GET, HEAD, OPTIONS
  2. Read X-CSRF-Token header
  3. Validate against CsrfTokenService.computeFor(sessionToken)
  4. Throw 400 if mismatch
```

### `RateLimitingGuard`

```
canActivate(context):
  1. Check THROTTLE_ENABLED (env)
  2. Check RATE_LIMIT_EXEMPT_IPS list
  3. Lookup rate_limit_entries for (ip, endpoint) in current window
  4. Throw 429 with Retry-After header if over limit
  5. Increment counter
```

## 16. IAM Context — Auth Services (Flows)

### `PasswordAuthService`

**`login(dto, deviceInfo)`:**

```
1. Sanitize email (lowercase, trim)
2. AuthUsersRepository.findByEmailWithPassword(email)
3. If not found: run bcrypt on dummy hash (timing attack prevention)
4. Check isBlocked → throw 401
5. Check accountLockedUntil > now → throw 401
6. PasswordService.compare(password, hash)
   If mismatch:
     - increment failedLoginAttempts
     - if >= MAX_FAILED_LOGIN_ATTEMPTS (5): set accountLockedUntil = now + 15 min
     - throw AUTH_INVALID_CREDENTIALS
7. AuthUsersRepository.resetAndRecordLogin(userId)
   - failedLoginAttempts = 0, accountLockedUntil = null
   - loginCount++, lastLoginAt = now
8. AuditService.logLogin(userId, deviceInfo)
9. executeAuthFlow(user, deviceInfo, sessionService, tokenService)
```

**`register(dto, deviceInfo)`:**

```
1. Sanitize email, firstName, lastName
2. Check email taken → throw AUTH_EMAIL_TAKEN
3. Hash password: bcrypt.hash(password, 12)
4. Generate iamUserId: crypto.randomUUID()
5. Transaction:
   a. INSERT into users
   b. INSERT into user_auth_provider (email + hashed password)
   c. Assign initial role:
      - If no SUPER_ADMIN exists → SUPER_ADMIN
      - Else → default role (CUSTOMER or configured)
6. AuditService.logRegistration(userId, ...)
7. executeAuthFlow(user, deviceInfo, sessionService, tokenService)
```

3. Return AuthResponseEnvelope
```

## 17. IAM Context — Auth Services (Token)

### `TokenService`

**`createAccessToken(payload)`:**
- Signs RS256 JWT with current private key
- `kid` = SHA-256 thumbprint of public key (for key rotation verification)
- Claims: `sub` (userGuuid), `jti` (UUID), `roles`, `storeId`, `iat`, `exp`

**`createTokenPair(userGuuid, sessionToken, roles, ...)`:**
- Access token: RS256 JWT, `JWT_ACCESS_TTL_SECONDS` (default 900 = 15 min)
- Refresh token: `crypto.randomBytes(32).toString('hex')` — 64-char hex string
- Refresh hash: `SHA-256(refreshToken)` — stored in DB (never plaintext)

**`buildAuthResponse(user, token, expiresAt, sessionGuuid, tokenPair?, cachedPermissions?)`:**

Returns complete `AuthResponseEnvelope`:
```typescript
{
  user: { guuid, iamUserId, firstName, lastName, email, emailVerified, phoneNumber, phoneNumberVerified },
  auth: {
    sessionId: sessionGuuid,
    sessionToken: token,    // null for web (set in cookie instead)
    tokenType: 'Bearer',
    expiresAt,              // session expiry
    refreshToken,
    refreshExpiresAt,
    accessToken,
  },
  context: {
    defaultStoreGuuid,      // from user.defaultStoreFk lookup
  },
  sync: {
    cursor: '0:0',          // initial sync cursor
    lastSyncedAt: null,
    deviceId,               // echoed from X-Device-Id header
  },
  offline: {                // null for web clients
    token: offlineJwt,      // RS256 JWT, 3-day TTL
    sessionSignature: hmac, // HMAC-SHA256 of offline payload
  } | null,
}
```

### `TokenLifecycleService`

**`refreshAccessToken(refreshToken, deviceId)`:**

```
1. Hash refresh token: SHA-256(refreshToken)
2. SessionsRepository.findByRefreshTokenHash(hash)
3. Validate:
   - Session exists
   - refreshTokenExpiresAt > now
   - refreshTokenRevokedAt is null (not already rotated)
4. Check for token reuse (theft detection):
   - If session.jti found in jti_blocklist → TOKEN_REUSE event
   - Terminate ALL sessions for this user (assume account compromise)
   - AuditService.logTokenTheftDetected(userId)
   - Throw AUTH_REFRESH_TOKEN_INVALID
5. Create new session (token rotation):
   a. New random session token
   b. New refresh token + hash
   c. INSERT new user_session row
   d. UPDATE old session: refreshTokenRevokedAt = now, revokedReason = 'ROTATION'
6. Check permissionsChanged:
   - Compare session.permissionsVersion vs user.permissionsVersion
   - If different: set permissionsChanged = true in response
7. Build auth response
```

## 19. IAM Context — Auth Services (OTP)

### `OtpService`

**`sendOtp(dto)`:**
```
1. OtpRateLimitService.checkAndRecordRequest(phone)
   → Throws 429 if > OTP limit
2. Msg91Service.sendOtp(phone)
3. Assert response.type !== 'error'
4. OtpRepository.insertOtpRecord(phone, 'PHONE_VERIFY', expiresAt, reqId)
5. Return { reqId, mobile: phone }
```

**`verifyOtp(dto)`:**
```
1. OtpRepository.findByIdentifierAndReqId(phone, reqId, 'PHONE_VERIFY')
2. Validate: isUsed=false, expiresAt > now
3. Msg91Service.verifyOtp(reqId, otp)
   If failure: OtpRateLimitService.trackVerificationFailure(phone)
4. OtpRepository.markAsUsedByReqId(reqId)
5. Return { verified: true, phone }
```

> **OTP Hashing:** HMAC-SHA256, not bcrypt. OTPs are short-lived and rate-limited; bcrypt overhead (100-300ms) has no security benefit here. RFC 6238 (TOTP) uses HMAC.

## 31. Session Rotation & Token Theft Detection

### Session Rotation (Cookie-Based Clients)

```
Every 1 hour (ROTATION_INTERVAL_SECONDS = 3600):

AuthGuard detects: now - session.lastRotatedAt > 3600s
  1. crypto.randomUUID() → newToken
  2. now + 30 days → newExpiresAt
  3. AuthContextService.rotateSessionToken(oldToken, newToken, newExpiresAt)
     → UPDATE user_session SET token=newToken, expiresAt=newExpiresAt, lastRotatedAt=now
  4. Set-Cookie: nks_session=newToken; maxAge=30d; httpOnly; sameSite=strict; secure

Effect: Stolen cookie valid for max 1 hour before legitimate user's next request invalidates it.
Mobile (Bearer token) is exempt — uses explicit refresh-token flow.
```

### Refresh Token Theft Detection

```
User A logs in on device X → gets refreshToken_1
Attacker steals refreshToken_1

User A calls POST /auth/refresh-token (legitimate use):
  → Creates new session, marks old session refreshTokenRevokedAt = now

Attacker calls POST /auth/refresh-token with refreshToken_1:
  → Found: session row with refreshTokenRevokedAt set AND jti in blocklist
  → TokenLifecycleService detects TOKEN_REUSE
  → Action: Terminate ALL sessions for this user
  → AuditService.logTokenTheftDetected(userId)
  → Response: AUTH_REFRESH_TOKEN_INVALID (401)

User A is now logged out everywhere → forced to re-authenticate
```

## 32. Key Security Features

| Feature | Implementation |
|---------|---------------|
| Default-deny auth | `AuthGuard` global; `@Public()` is the opt-out |
| CSRF protection | Double-submit cookie pattern, HMAC binding (`CsrfMiddleware` + `CsrfGuard`) |
| Session rotation | 1-hour cookie expiry window, new token on each rotation |
| Token theft detection | Refresh reuse → terminate all sessions |
| IP fingerprinting | HMAC-SHA256(IP, secret) stored per session, change logged |
| Rate limiting | Per-IP throttle (global), OTP limits (100/24h), login limits (5/15m) |
| Account lockout | 5 failed logins → 15 min lockout |
| Password hashing | bcrypt 12 rounds |
| OTP hashing | HMAC-SHA256 (RFC 6238 design) |
| JTI blocklist | Revoked JWT IDs stored with TTL in DB |
| Helmet headers | CSP, HSTS, X-Frame-Options=deny, referrer policy |
| Audit logging | All auth events logged async (EventEmitter, off hot path) |
| Permission cache invalidation | Explicit on role/permission changes (O(k) via secondary index) |
| Offline session HMAC | Mobile stores signature, server re-validates on every sync push |
| Email enumeration prevention | Timing guard: dummy bcrypt when user not found |
| Device binding | deviceType, appVersion, userAgent per session |
| Signed cookies | `cookie-parser` with signing secret |
| RS256 JWT | Asymmetric signing; public key shareable for offline verification |
| Key rotation | `KeyRotationScheduler` daily cron, 30-day grace period |
| LIKE injection | `escapeLike()` in query-helpers escapes `%`, `_`, `\` before binding |

## 33. Scheduled Jobs (Cron)

| Service | Schedule | Action |
|---------|----------|--------|
| `SessionCleanupService` | `30 0 * * *` (00:30 UTC) | `cleanupExpiredSessions()` + `cleanupOldRevokedSessions()` + `cleanupExpiredDeviceRevocations()` |
| `KeyRotationScheduler` | `0 2 * * *` (02:00 UTC) | Check key age, alert if rotation needed. Also runs on startup. |

All cleanup tasks inside `SessionCleanupService` run via `Promise.allSettled()` — one failure doesn't block others.

## 34. Event System

Using `@nestjs/event-emitter`:

| Event | Emitter | Listener |
|-------|---------|---------|
| `AuditEvents.LOG` | `AuditService` (all auth/domain actions) | `AuditEventListener` → INSERT audit_logs |
| `AuditEvents.LOGGED` | `AuditEventListener` | — (completion signal) |
| `SessionRevocationEvent` | `SessionService.invalidateSessionByToken()` | `SessionRevocationListener` → optionally terminate other sessions |

**Design intent:** All audit writes are async and off the hot request path. A slow DB write does not delay the API response.
